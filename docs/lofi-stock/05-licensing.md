# 05 — Licensing & Attribution

Pixabay-specific licence rules and how we record/honor them.

## Pixabay Content License (summary)

As of 2026-05 (verify at https://pixabay.com/service/license-summary/ before launch):

- ✅ Free for commercial use
- ✅ No attribution required (but appreciated)
- ❌ Cannot sell or redistribute on similar platforms
- ❌ Cannot use content of identifiable people in a defamatory way
- ❌ Cannot imply endorsement by Pixabay or its contributors

For YouTube videos: fully OK. Monetization OK. Distribution OK.

## What we record per asset

For each picked track, snapshot the licence string at pick time into `lofiAssets.sourceLicence`. Reason: Pixabay can update its terms; if a future audit requires "what licence did this content use when you embedded it?", we have the answer per-track.

Stored shape:
```
sourceLicence = "Pixabay Content License (2026-05) — commercial use OK, no attribution required, no redistribution"
```

Don't store URL to licence page — text snapshot is the contract proof.

## Attribution policy

Pixabay does not require attribution. But:

- **In-video credit:** optional, configurable per user. Default OFF. If user opts in, render a 2-second "Music from Pixabay" overlay in last second of video. ffmpeg drawtext filter.
- **Video description:** auto-generate suggested description with track titles + Pixabay credit when (post-MVP) we add YouTube auto-upload.

`lofiVideos` new column (post-MVP):
```ts
includePixabayCredit: boolean('include_pixabay_credit').notNull().default(false)
```

For MVP: skip credit overlay, document in user-facing copy ("Music sourced from Pixabay — free for commercial use, no attribution required").

## Identifiable persons constraint

Pixabay images/videos sometimes include identifiable people. Not relevant for music — Pixabay music is instrumental. No action.

## Future provider licence handling

If we add Freesound:
- Per-track licence varies (CC0, CC-BY, CC-BY-NC, etc.)
- Must filter out CC-BY-NC (non-commercial) automatically since users may monetize
- Must render attribution overlay for CC-BY tracks (mandatory)
- `sourceAttribution` column will carry the required credit string

Architecture supports it (`sourceAttribution` already in schema) but UX gets more complex. Defer to post-MVP.

## Terms of service compliance

Pixabay ToS forbids:
- Mass automated downloads (we don't — only download when user submits a video)
- Re-uploading their content as our own (we link to their CDN, don't re-host)
- API key sharing (server-side only, never exposed to client)

Our usage pattern stays compliant.

## User-facing licence display

In `LofiStockForm` selection drawer, each track has small ℹ tooltip showing:
```
Pixabay Content License
Free for commercial use · No attribution required
[ View license ↗ ]
```

Link opens https://pixabay.com/service/license-summary/ in new tab.

## Per-video licence summary

In `LofiVideoView` after render complete, show licences-used section:
```
Music sources
  18 tracks from Pixabay (Pixabay Content License)

Visual sources  
  1 image generated by you (AI)
```

Useful for users who upload to YouTube and want to fill out music attribution section confidently.
